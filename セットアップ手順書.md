# セットアップ手順書｜社内向け

このドキュメントは **初めてセットアップする人** 向けです。所要約45〜60分。

## 全体像

```
[Apify]                  → リール収集（10分）
[OpenAI API]             → AI分析（10分）
[Google Cloud]           → スプシ書き込み権限（20分・最も手間）
[Google スプレッドシート] → 結果出力先（5分）
[ローカル設定]           → 環境変数・コード設定（10分）
```

セットアップで取得・配置するもの：

1. Apify トークン（30〜45文字）
2. OpenAI API キー（`sk-` 始まり）
3. Google Cloud サービスアカウントの鍵JSON
4. Google スプレッドシートのID

これらを `~/.zshrc` と `gsheets.py` に書き込んだら完成。

---

## Phase 0: 環境準備

ターミナルで以下を実行：

```bash
# Pythonパッケージをインストール（既に他のプロジェクトで入ってる場合はスキップでOK）
pip3 install --break-system-packages openpyxl gspread google-auth requests openai openai-whisper

# ffmpeg をインストール
brew install ffmpeg

# 認証JSONを置く場所を準備
mkdir -p ~/.config/gcloud

# Whisper の medium モデルを事前ダウンロード（約1GB、初回のみ）
python3 -c "import whisper; whisper.load_model('medium')"
```

確認：
```bash
which ffmpeg && python3 -c "import whisper; print('whisper OK')"
```

## Phase 1: Apify トークン取得（10分・無料枠あり）

**Apify** は Web スクレイピングのプラットフォーム。Instagram からリール情報を収集するのに使う。月額 **$5の無料枠** あり。

### 手順

1. <https://apify.com/> にアクセス → **Sign up**（Google認証が楽）
2. ログイン後、左サイドバー下部の **Settings**（歯車） → **Integrations**
   - 直接URL：<https://console.apify.com/settings/integrations>
3. **Personal API tokens** セクションで **+ Create a new token**
4. 入力：
   - Description：`ig-reel-research`
   - Set expiration date：OFF
   - Limit token permissions：OFF
5. **Create** をクリック → 表示されたトークンをコピー（**1度しか表示されない**ので必ずメモ）

> [📸 スクショ: `screenshots/1_apify_token.png` — Apifyのトークン作成画面]

## Phase 2: OpenAI APIキー取得（10分・要クレカ）

### 手順

1. <https://platform.openai.com/> へ（`chatgpt.com` ではない）
2. Sign up / Log in
3. **Settings → Billing**
   - <https://platform.openai.com/settings/organization/billing/overview>
4. クレカ登録 → **Add to credit balance** で **$5以上** チャージ
   - Auto recharge は **OFF推奨**
5. **API keys**
   - <https://platform.openai.com/api-keys>
6. **+ Create new secret key**
   - Name：`ig-reel-research`
   - Project：Default project
   - Permissions：**All**
7. キーをコピー（`sk-proj-XXXX...`、**1度しか表示されない**）

> [📸 スクショ: `screenshots/2_openai_key.png` — OpenAI APIキー一覧画面]

## Phase 3: Google Cloud サービスアカウント（20分・最も手間）

ここが最大の難所。 **「サービスアカウント」 = ロボット用のGoogleアカウント** で、Pythonスクリプトがスプシに書き込めるようにする。

### 3-1. プロジェクト作成

1. <https://console.cloud.google.com/> にログイン
2. 上部のプロジェクトドロップダウン → **新しいプロジェクト**
3. プロジェクト名：`ig-reel-research-<あなたの名前>`（一意になるように）
4. 場所：「組織なし」のまま
5. 作成

> [📸 スクショ: `screenshots/3a_gcp_project_create.png`]

### 3-2. API有効化

1. ハンバーガーメニュー → **APIとサービス → ライブラリ**
2. 検索 `Google Sheets API` → **有効にする**
3. 同じく `Google Drive API` → **有効にする**

> [📸 スクショ: `screenshots/3b_gcp_apis_enabled.png`]

### 3-3. サービスアカウント作成

1. ハンバーガーメニュー → **IAMと管理 → サービスアカウント**
2. 上部 **+ サービスアカウントを作成**
3. 名前：`sheets-writer`
4. 「作成して続行」
5. ロール：**基本 → 編集者** を選択
6. 「続行」→「完了」

> [📸 スクショ: `screenshots/3c_service_account_role.png`]

### 3-4. 鍵JSONダウンロード

1. 作成したサービスアカウントの行をクリック
2. **キー** タブ → **鍵を追加 → 新しい鍵を作成**
3. JSON を選択 → **作成**
4. ファイルが自動DL（`xxx-yyyy.json` のような名前）

> [📸 スクショ: `screenshots/3d_create_key_json.png`]

### 3-5. JSONを所定位置へ

ターミナルで：

```bash
# Downloadsから移動（実際のファイル名に置換）
mv ~/Downloads/プロジェクト名-xxxxx.json ~/.config/gcloud/sheets-writer.json

# パーミッションを600に（自分のみ読み書き可）
chmod 600 ~/.config/gcloud/sheets-writer.json
```

### 3-6. サービスアカウントのメールをコピー

JSONの `client_email` 項目（`sheets-writer@xxx.iam.gserviceaccount.com`）をメモ。Phase 4 で使う。

```bash
# ターミナルで確認
python3 -c "import json; print(json.load(open('/Users/$(whoami)/.config/gcloud/sheets-writer.json'))['client_email'])"
```

## Phase 4: Google スプレッドシート準備（5分）

1. <https://sheets.google.com/> で **空白のスプレッドシート** を新規作成
2. タイトル：`IG Reel Research`（任意）
3. 右上 **共有** → Phase 3-6 のサービスアカウントメールを貼り付け
4. 権限：**編集者**
5. **通知を送信** OFF（サービスアカウントは人ではないので）
6. 共有
7. URL からスプシIDをコピー：
   ```
   https://docs.google.com/spreadsheets/d/【ここをコピー】/edit
   ```

## Phase 5: コード配置と環境変数設定（10分）

### 5-1. スキルを所定位置へ配置

このパッケージの `ig-reel-research/` フォルダを `~/.claude/skills/` にコピー：

```bash
# このパッケージを展開した場所から
cp -r ig-reel-research ~/.claude/skills/

# 確認
ls ~/.claude/skills/ig-reel-research/
```

### 5-2. スプシIDをコードに埋め込み

`~/.claude/skills/ig-reel-research/scripts/gsheets.py` の16行目を編集：

```python
# 変更前
DEFAULT_SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID_HERE'

# 変更後（Phase 4 で取得したIDに置換）
DEFAULT_SPREADSHEET_ID = '1AbC2dEfGhIjK_XyZ123456789...'
```

### 5-3. 環境変数を `.zshrc` に追記

```bash
open -a "TextEdit" ~/.zshrc
```

ファイル末尾に以下を追記（実際の値に置換）：

```bash
# ig-reel-research
export APIFY_TOKEN="apify_api_xxxxxxxxxxxxxxxxxxxx"
export OPENAI_API_KEY="sk-proj-xxxxxxxxxxxxxxxxxxxx"
export GOOGLE_SHEETS_CREDENTIALS="$HOME/.config/gcloud/sheets-writer.json"
```

保存（Cmd+S）してターミナルで反映：

```bash
source ~/.zshrc
echo "APIFY=${#APIFY_TOKEN}文字 / OPENAI=${#OPENAI_API_KEY}文字 / JSON=$GOOGLE_SHEETS_CREDENTIALS"
```

期待出力例：
```
APIFY=46文字 / OPENAI=164文字 / JSON=/Users/yourname/.config/gcloud/sheets-writer.json
```

## トラブルシュート

### `[FATAL] APIFY_TOKEN が短すぎます`
→ `.zshrc` の値が空 or 文字数不足。`source ~/.zshrc` 実行 + 値を確認。

### `[FATAL] GOOGLE_SHEETS_CREDENTIALS が見つかりません`
→ JSONのパスが違う。`ls $GOOGLE_SHEETS_CREDENTIALS` で存在確認。

### `gspread.exceptions.APIError: PERMISSION_DENIED`
→ スプシをサービスアカウントに編集者で共有していない（Phase 4-3）。スプシURL の「共有」を再確認。

### Whisper が遅い・落ちる
→ `IG_REEL_DISABLE_TRANSCRIBE=1 python3 -u run_production_research.py キーワード` で文字起こしをスキップ可。フックの精度は落ちる。

### `[FATAL] 別の ig-reel-research プロセス (PID xxx) が実行中`
→ 前回の実行が異常終了。`rm /tmp/ig-reel-research.lock` で解除。
