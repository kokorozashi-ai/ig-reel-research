セットアップが完了したら、ここから日常的な使い方を覚えればOK。

## 基本：Claude Code で自然言語で頼む

すべて **チャットで普通に日本語** で頼むだけ。

### キーワードリサーチ

```
ダイエットでリサーチして
```

→ Claude が条件を聞く：
```
リサーチを始めるね！以下を教えて：

- キーワード（必須）：何を調べる？
- 期間：7日 / 14日（既定） / 30日 / 90日
- 再生回数の条件：
  - 絶対値: 5万以上（既定）/ 10万以上 / 30万以上 など
  - フォロワー比: 5倍以上 / 10倍以上 など

特になければキーワードだけ教えてくれれば、デフォルト（14日間・5万再生以上）で実行するよ！
```

→ ユーザーが「ダイエット」だけ返答 → 自動実行（15〜30分）→ スプシに新タブ追加

### バリエーション

| 発話 | 動作 |
|---|---|
| `ダイエット` | 14日・5万再生以上 |
| `ダイエットで30日、10万再生以上` | 30日・10万以上 |
| `ダイエットでフォロワー5倍以上` | 14日・フォロワー比5倍 |
| `美容と節約で2週間` | 2キーワード・14日 |

## ウォッチリスト機能

「定点観測したい」アカウントを登録できる。リサーチ結果から見つけた良いアカウントや、業界の有名アカウントなど。

### 登録方法（2通り）

#### 方法A：チャットで頼む

```
@tomoya_tore @rinochan.diet をウォッチリストに追加（ジャンル：フィットネス、メモ：編集が上手い）
```

→ Claude が `add_to_watchlist.py` を裏で実行。

#### 方法B：スプシ直接編集

スプシの「ウォッチリスト」タブを開いて、行を追加：

| No. | アカウント名 | ジャンル | 注目ポイント | 登録日 | メモ |
|---|---|---|---|---|---|
| 1 | tomoya_tore | フィットネス | テロップが上手い | 2026-05-04 | |

> [📸 スクショ: `screenshots/4_ウォッチリスト登録.png`]

### ウォッチリストモードでの分析

```
ウォッチリストで分析して
```

→ 登録された全アカウントの直近期間内のリールを取得・分析 → 「ウォッチリスト YYYYMMDD」タブに書き込み。
※ ウォッチリストは再生回数フィルタを **適用しない**（学習用なので全リール対象）。

### キーワードリサーチでの★マーカー

普通のキーワードリサーチで、登録済みアカウントのリールが含まれていたら、 **そのリール列のヘッダーに ★ + 紫背景** で強調表示される。

## 自動実行（定点観測）

毎月1日と15日の朝9時に **ウォッチリスト分析** を自動実行する仕組みを用意。

### 有効化

このパッケージに含まれる `launchd-template.plist` を編集して配置：

```bash
# テンプレートを編集（YOUR_USERNAME を実際のユーザー名に置換）
cp launchd-template.plist ~/Library/LaunchAgents/com.YOURUSER.ig-watchlist.plist
# テキストエディタで開いて YOUR_USERNAME を sed で置換
sed -i '' "s/YOUR_USERNAME/$(whoami)/g" ~/Library/LaunchAgents/com.YOURUSER.ig-watchlist.plist

# 登録
launchctl load -w ~/Library/LaunchAgents/com.YOURUSER.ig-watchlist.plist

# 確認
launchctl list | grep ig-watchlist
```

詳細は `launchd-template.plist` 内のコメントを参照。

### 一時停止 / 再開

```bash
# 停止（次回まで実行されない）
launchctl unload ~/Library/LaunchAgents/com.YOURUSER.ig-watchlist.plist

# 再開
launchctl load -w ~/Library/LaunchAgents/com.YOURUSER.ig-watchlist.plist

# 完全削除
launchctl unload ~/Library/LaunchAgents/com.YOURUSER.ig-watchlist.plist
rm ~/Library/LaunchAgents/com.YOURUSER.ig-watchlist.plist
```

### テスト実行（手動で発火）

```bash
launchctl start com.YOURUSER.ig-watchlist
```

### ログ確認

```bash
tail -100 ~/.cache/ig-reel-research/cron.stdout.log
tail -100 ~/.cache/ig-reel-research/cron.stderr.log
```

## よくある問題

### Q. 実行後にスプシに何も書かれていない

→ ログを確認：
```bash
tail -50 ~/.cache/ig-reel-research/cron.stdout.log
```

### Q. 「リサーチして」と言ったらすぐ実行される

→ SKILL.md の「ヒアリング先行ルール」が効いていない。Claude Code を再起動。

### Q. Whisperが遅い（30分以上かかる）

→ M1以外のIntel Macでは medium モデルが厳しい。 `--whisper-model small` 相当の対応や `IG_REEL_DISABLE_TRANSCRIBE=1` でスキップ可。

### Q. APIコストが心配

→ OpenAI のダッシュボード（<https://platform.openai.com/usage>）で利用量を確認。月額上限を設定可。

### Q. Apify がエラーを返す

→ `Apify` の Console（<https://console.apify.com/runs>）で実行ログを確認。Instagram のスクレイパー側で一時的に動かないことがある（数日待てば直る）。

### Q. ロックファイルで起動できない

```
[FATAL] 別の ig-reel-research プロセス (PID xxx) が実行中
```

→ 前回が異常終了。
```bash
rm /tmp/ig-reel-research.lock
```

## メンテナンス

### スキルを更新したい

オリジナルから新しい版を受け取ったら：

```bash
# スキルをいったん削除
rm -rf ~/.claude/skills/ig-reel-research

# 新しい版をコピー
cp -r 新しいパッケージ/ig-reel-research ~/.claude/skills/

# スプシIDを再設定（毎回これだけは必要）
# scripts/gsheets.py の16行目 DEFAULT_SPREADSHEET_ID を自分のIDに
```

### スプシを別のものに変えたい

`~/.claude/skills/ig-reel-research/scripts/gsheets.py` の `DEFAULT_SPREADSHEET_ID` を新しいIDに置換。新しいスプシのサービスアカウント共有も忘れずに。

### ウォッチリストアカウントを削除

スプシの「ウォッチリスト」タブで該当行を削除（CLI削除コマンドは未実装）。
