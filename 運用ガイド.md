# 運用ガイド｜日常的な使い方

セットアップが完了したら、ここから日常的な使い方を覚えればOK。

---

## 基本：Claude Code に話しかけるだけ

すべて **チャットで普通に日本語** で頼むだけ。コマンドを覚える必要はありません。

### キーワードでリサーチ

```
ダイエットでリサーチして
```

→ AI から条件のヒアリングがあります：
```
リサーチを始めるね！以下を教えて：

- キーワード（必須）：何を調べる？
- 期間：7日 / 14日（既定）/ 30日 / 90日
- 再生回数の条件：
  - 5万以上（既定）/ 10万以上 / 30万以上 など
  - フォロワー比 5倍以上 / 10倍以上 など

特になければキーワードだけ教えてくれれば、デフォルト（14日間・5万再生以上）で実行するよ！
```

→ **キーワードだけ答える** か、 **条件を細かく指定する** か、お好みでどうぞ。

### 話しかけ方の例

| あなたの一言 | 動作 |
|---|---|
| `ダイエット` | 14 日間・5 万再生以上で調査（既定） |
| `ダイエットで 1 ヶ月、10 万再生以上で` | 30 日・10 万再生以上 |
| `美容で 2 週間` | 14 日 |
| `副業で 3 ヶ月、フォロワーの 5 倍以上で` | 90 日・フォロワー比 5 倍以上 |
| `節約と料理で` | 2 キーワード・14 日（複数キーワード可）|

## ウォッチリスト機能

「**この人のリールは継続的にチェックしたい**」というアカウントを登録できる機能です。

### 何のために使う？

- 業界で参考にしたい上手なアカウントを **定期観察** したい
- 自分の競合を **継続トラッキング** したい
- ベンチマークしたい人の **新作リール** を毎月見たい

### アカウントを登録する 2 つの方法

#### 方法 A：チャットで頼む

```
@tomoya_tore @rinochan.diet をウォッチリストに追加して
（ジャンル：フィットネス、メモ：編集が上手い）
```

→ AI が裏で登録処理してくれます。

#### 方法 B：スプシで直接編集

スプレッドシートの「**ウォッチリスト**」タブを開いて、行を追加：

| No. | アカウント名 | ジャンル | 注目ポイント | 登録日 | メモ |
|---|---|---|---|---|---|
| 1 | tomoya_tore | フィットネス | テロップが上手い | 2026-05-04 | |

> 📸 [スクショ: `screenshots/4_ウォッチリスト登録.png`]

### ウォッチリストモードでの分析

```
ウォッチリストで分析して
```

→ 登録された全アカウントの直近の最新リールを取得・分析してくれます。
※ ウォッチリストは **再生数の制限なし** で全リールが対象（学習用なので）

### ★ マークの意味

普通のキーワードリサーチをした時、 **登録済みアカウントが結果に含まれていれば** スプシでそのリールの列に **★** が紫色の背景で表示されます。

> 📸 [スクショ: `screenshots/5_star_marker.png`]

「あ、ウォッチしている人がこのキーワードで伸びてる」 が一目で分かります。

## 自動でウォッチリストをチェックする（任意）

毎月 1 日と 15 日の朝 9 時に **ウォッチリスト分析を自動実行** する仕組みを用意しています。

### 有効にする

このプロジェクトに含まれる `launchd-template.plist` を編集して配置：

ターミナルで：

```bash
# テンプレートをコピー（YOUR_USERNAME はあなたのユーザー名に置き換え）
cp launchd-template.plist ~/Library/LaunchAgents/com.YOUR_USERNAME.ig-watchlist.plist

# テンプレート内の YOUR_USERNAME を実ユーザー名に置換
sed -i '' "s/YOUR_USERNAME/$(whoami)/g" ~/Library/LaunchAgents/com.YOUR_USERNAME.ig-watchlist.plist

# 登録
launchctl load -w ~/Library/LaunchAgents/com.YOUR_USERNAME.ig-watchlist.plist

# 確認
launchctl list | grep ig-watchlist
```

詳しい使い方は `launchd-template.plist` のファイル内コメントに書いています。

### 一時停止 / 再開

```bash
# 停止（次回まで実行されない）
launchctl unload ~/Library/LaunchAgents/com.YOUR_USERNAME.ig-watchlist.plist

# 再開
launchctl load -w ~/Library/LaunchAgents/com.YOUR_USERNAME.ig-watchlist.plist

# 完全削除
launchctl unload ~/Library/LaunchAgents/com.YOUR_USERNAME.ig-watchlist.plist
rm ~/Library/LaunchAgents/com.YOUR_USERNAME.ig-watchlist.plist
```

### 試しに 1 回だけ動かしたい

```bash
launchctl start com.YOUR_USERNAME.ig-watchlist
```

### ログを見たい

```bash
# 標準ログ
tail -100 ~/.cache/ig-reel-research/cron.stdout.log

# エラーログ
tail -100 ~/.cache/ig-reel-research/cron.stderr.log
```

## スプレッドシートの読み方

### タブの種類

| タブ | 役割 | 自動更新 |
|---|---|---|
| **使い方ガイド** | このツールの仕様説明 | リサーチごとに更新 |
| **ウォッチリスト** | 登録アカウント一覧（手動編集OK）| **絶対に消えない** |
| **<キーワード名>** | リサーチ結果のスナップショット | 累積（手動で消すまで残る）|
| **ウォッチリスト YYYYMMDD** | ウォッチリスト分析結果 | 累積 |

### 結果タブの見方

> 📸 [スクショ: `screenshots/2_リサーチ結果タブ.png`]

```
        |  A             |  B      |  C      |  D      |
--------|----------------|---------|---------|---------|
 1      | 【サマリー】     | （結合セルでサマリー本文：実施日・キーワード・期間・トレンド・アクション）|
 2      | 属性            | リール1  | リール2  | ★リール3 |  ← ★ はウォッチ中アカウント
 3      | リールURL       | https.  | https.  | https.  |
 4      | アカウント名     | @user1  | @user2  | @user3  |
 5      | フォロワー数     | 12,400  | 8,500   | 5,200   |
 6      | 再生回数        | 152万    | 85万    | 42万    |
 ...    | （全 20 項目）   |         |         |         |
```

横方向にスクロールすれば各リールの情報が見えます。1 行目（サマリー）と 2 行目（ヘッダー）、左 1 列は **常に固定表示** されるので迷子になりません。

### 1 リールあたりの分析項目（20 項目）

| # | 項目 | 内容 |
|---|---|---|
| 1 | リール URL | 動画への直リンク |
| 2 | アカウント名 | クリックでプロフィールへ |
| 3 | フォロワー数 | 投稿時点 |
| 4 | 再生回数 | カンマ区切り |
| 5 | いいね数 | 〃 |
| 6 | コメント数 | 〃 |
| 7 | 投稿日 | YYYY-MM-DD |
| 8 | エンゲージメント率 | （いいね＋コメント）÷ 再生数 ×100 |
| 9 | 動画尺 | ○分○秒 |
| 10 | ジャンル | AI による自動分類 |
| 11 | ハッシュタグ | 使われていたタグ全部 |
| 12 | フック | 動画冒頭 3 秒の要約（音声 AI を最優先で判定）|
| 13 | 全文文字起こし | 動画音声を AI が文字に変換 |
| 14 | キャプション全文 | 投稿時のキャプション原文 |
| 15 | バズ仮説 | 「なぜ伸びたか」を AI が言語化 |
| 16 | 改善提案 | 「もっと伸ばすには」を AI が提案 |
| 17 | **BGM** | 使われている音源・曲名 |
| 18 | **動画カット数** | 編集の数（テンポの目安）|
| 19 | **シーン構成** | 各カットに何が映っているか AI が説明 |
| 20 | **テロップ OCR** | 画面上のテロップ文字を全部書き起こし |

## メンテナンス

### このツールの新バージョンが出たら

```bash
# 古いものを消す
rm -rf ~/.claude/skills/ig-reel-research

# 新しいものをコピー
cp -r 新しいフォルダ/ig-reel-research ~/.claude/skills/

# スプレッドシート ID を再設定（毎回必要）
sed -i '' "s|YOUR_SPREADSHEET_ID_HERE|あなたのID|" ~/.claude/skills/ig-reel-research/scripts/gsheets.py
```

### 使うスプレッドシートを変えたい

`~/.claude/skills/ig-reel-research/scripts/gsheets.py` の中のスプシ ID を新しい ID に置き換えてください。新しいスプシも忘れずに自動操作アカウント（サービスアカウント）と共有してください。

### ウォッチリストからアカウントを削除

スプシの「ウォッチリスト」タブで該当行を削除すれば OK。
